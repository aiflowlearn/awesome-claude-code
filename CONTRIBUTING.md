# Contributing

Thanks for improving awesome-claude-code. This repo values practical assets that users can copy, run, and learn from.

## What to Contribute

- CLAUDE.md templates for real project types.
- Claude Code hooks with clear safety behavior.
- Exercises with objectives, steps, and verification.
- LLM Wiki schemas and examples.
- Corrections to the Claude Code knowledge base.

## Template Guidelines

- Keep CLAUDE.md templates concise and enforceable.
- Prefer tables for rules and verification.
- Avoid secrets, private endpoints, and company-specific credentials.
- Keep files focused; split large examples.

## Exercise Guidelines

Each exercise should include:

1. Goal.
2. Context.
3. Steps.
4. Expected outcome.
5. Verification command or observation.

## Wiki Schema Guidelines

- Define directory structure.
- Define page fields.
- Explain linking rules.
- Include one example page when possible.

## Pull Request Process

1. Fork the repo and create a focused branch.
2. Add or update files with one clear intent.
3. Check licenses for the paths you changed.
4. Verify examples and commands where possible.
5. Open a PR explaining what changed and why.

By contributing, you agree that your contribution is licensed under the applicable license for the target directory.
